## C. 程式 Bug(潛在/已確認)

### C-0 ✅ 時間結算卡死 → 1-1 過關後進不了 1-2(2026-06-12 實測發現,已修復)
- `UpdateLevelClearTransition` 中 `actualDeduct = min(unitsToDeduct, int(m_TimeRemaining))`:遊玩時時間以 `dt×2.5` 扣減必帶小數,結算扣到剩 `0.x` 時 `int(0.x)=0` → 永遠扣不到 0,也不滿足 `<= 0` 的轉場條件,**無限卡在結算狀態**(HUD 顯示 0 但不換關)。
- 症狀(log 實證):分數每幀 +50 後突然停止,無 `Entered level clear pause` 記錄,遊戲仍在跑。
- 修復:殘餘小數(< 1 單位)視為 0,扣完整數後直接歸零進停頓。

### C-0b ✅ 大形態抵達新關卡會嵌進地板 → 開場即死(2026-06-12 實測發現,已修復)
- 關卡 JSON 的 `playerSpawn` y 值以**小馬力歐(16px 高)的左上角**為基準(例:y=192、地板頂 208)。`Player::SetSpawnPosition` 原本直接把「頭頂」放在 spawn y,SUPER/FIRE(32px 高)抵達時腳底變成 224,**嵌進地板 16px**。
- 嵌住後:落地判定(容差 0.5px)永遠不成立 → 側碰修正持續推擠 + 重力下沉 → 掉出 killZ,進關 0.4 秒即死。死後 `SetForm(SMALL)` 是「保底部」邏輯,腳底仍是 224,連死兩次,直到以 SMALL 形態死亡才恢復正常(log 實證:1-1 過關進 1-2 連噴兩命,進地下又噴一命 → GAME OVER)。
- 修復:`SetSpawnPosition` 改為**腳底對齊**(bottom = spawn y + TILE_SIZE),任何形態都站上同一格地面;對小馬力歐行為不變,所有 LoadLevel / 水管 targetSpawn / checkpoint 重生一體適用。

### C-0c ✅ 中繼點重生時敵人疊在玩家身上 → 重生即死循環(2026-06-12 實測發現,已修復)
- 死亡後從中繼點(1-1 的 x=2000)重生,進入 Playing 的**第一幀** `CheckEnemySpawnQueue` 以「敵人 x ≤ 鏡頭右緣」為條件,把 0~2192 整段路的 17 隻敵人一次全部生成——其中 x=1984、2016 的 Goomba 直接疊在玩家身上,0.01 秒內判死,無限循環直到 GAME OVER(log 實證)。
- 修復:中繼點重生時(僅此情境)修剪生成佇列,**鏡頭右緣以左的敵人不再生成**——這正是原版 NES midway 重生的行為(敵人只在其欄位從畫面右緣進入時生成)。一般開場與水管抵達**不修剪**,保留刻意設計在初始畫面內的敵人(如 1-2 水管口的食人花、地下室入口的 Goomba)。
- [GameManager.cpp:1309-1312](src/GameManager.cpp#L1309):`CheckShellEnemyCollision()` 結尾 `if (!shellKilledAny) m_ShellComboCount = 0;` **每一幀**只要殼沒殺到敵人就歸零。
- 殼滑行撞敵是一隻一隻撞(不同幀),所以 combo 在兩殺之間必被清零 → 每殺永遠都是第一階 100 分,200/400/…/1UP 的殼連殺表形同虛設。
- 修法:應在「殼停止滑行 / 殼被消滅 / 玩家再次踢殼」時重置,而不是每幀重置。

### C-0d ✅ 次像素腳尖懸空站立(2026-06-12 截圖實證,已修復)
- 落地判定(Pass 2A)只要求「水平重疊量 > 0」即可站立,玩家可用 <1px 的趾尖搭住方塊邊緣,看起來整個人懸在半空中(實證:1-2_ground_2 水管與樓梯之間,腳底懸在 y=176 高度)。
- 原版 NES 的支撐是兩個內縮約 2px 的腳部取樣點,不會發生這種情況。
- 修復:落地新增 `MIN_SUPPORT_OVERLAP = 3.0f`,水平重疊不足 3px 不算踩到;同時削弱「貼牆跳」(貼牆下落時側推殘留的小量重疊不再被誤判為地板)。

### C-0e ✅ 鑽出水管動畫永遠不結束 → 玩家凍結在管口(2026-06-12 實測發現,已修復)
- `EnteringPipe` 播完有 `IsAnimationFinished() → ChangeLevel` 接手,但 **`ExitingPipe` 播完後沒有任何程式碼切回 `Normal`**;UpdatePlaying 對該狀態只更新鏡頭與畫面,玩家永遠凍在管口、時間也停止。
- 觸發條件:換關目的地 `targetSpawn` 落在 EnterablePipe 內 → ChangeLevel 觸發鑽出動畫(此路徑先前從未被任何關卡資料走到,直到 1-2 獎勵房的回程 spawn 設在水管內才引爆)。
- 修復:`Player::FinishPipeExit()`(回 Normal、清速度、重判落地),UpdatePlaying 在 `ExitingPipe && IsAnimationFinished()` 時呼叫。

### C-2 🟡 無目標水管會吞掉同幀其他水管的判定
- [GameManager.cpp:1066-1070](src/GameManager.cpp#L1066):`CheckPipeTransition()` 掃描中遇到「可進入但 targetLevel 為空」的水管時直接 `return false`,**中斷整個函式**,排在它後面的合法水管當幀不會被檢查。

### C-3 🟡 死亡敵人、掉出地圖的道具永遠留在容器中
- 敵人:`SetAlive(false)` 後仍留在 `m_Enemies`([GameManager.cpp:399-404](src/GameManager.cpp#L399)),每幀仍被迴圈+`dynamic_cast` 掃過,直到換關才清空。
- 道具:`m_Items` 只移除 `Collected` 狀態;蘑菇/星星走出關卡邊緣掉入虛空後**沒有 Kill Z 判定**,會永遠留著並持續 Update(位置 y 無限增大)。
- 影響:長關卡的效能與記憶體緩慢劣化;不會立即出錯。

### C-4 🟡 與敵人重疊時跳躍會誤判成踩踏(Known_isseues/global.md 已記錄)
- [GameManager.cpp:1191-1197](src/GameManager.cpp#L1191):`fallingOntoEnemy` 只要 `pVel.y > 0`(往下)+ 前一幀腳底在敵頭上方 6px 內即成立;與敵人重疊起跳後的下落幀容易滿足,形成「貼著敵人跳=免傷踩殺」。

### C-5 🟡 貼牆跳 / 貼牆敲磚(Known_isseues/global.md 已記錄)
- `CheckBlockCollision` 的 Pass 2A 落地判定(`SUPPORT_TOLERANCE 0.5px`)與 Pass 2B 側推的互動仍有邊界案例:貼牆下落時側推每幀把玩家擠出,腳底跨過某格牆磚頂邊時會被判落地一幀 → 可跳;Pass 1 的 `bestOverlapX` 也可能選到玩家側面貼著的牆磚。
- 已知修法方向(global.md):改為只檢查玩家鄰近格的方塊,並以 `penY < penX` 約束軸向。

### C-6 ✅ 樹平台可以從下方穿越(2026-06-12 修復:改為正式單向平台)
- 原版的樹平台/升降梯本來就是 semisolid:從下方穿過是正確行為,問題在舊實作 `IsSolid()=true` 又靠容差「偶爾」擋住,卡在兩種行為之間。
- 修復:`Block` 新增 `IsOneWay()` 虛擬方法;`TreePlatformBlock` 與 `MovingPlatformBlock` 回傳 true。玩家碰撞 Pass 1(頂磚)與 Pass 2B(側推)跳過單向平台,Pass 2A 的落地條件(`vel.y >= 0` + 從上方跨越頂面)天然只允許從上方站上。敵人/道具/火球仍視其為實心(紅龜巡邏樹平台不受影響)。
- 同批修復(2026-06-12):
  - **verticalWrap 循環平台不像原版**:1-2 的循環範圍只有 64px 且全程在畫面內,瞬移當場可見;且 wrap 瞬移量計入 `m_FrameDelta`,站在上面的玩家會被一起傳送。修復:瞬移幀的 `m_FrameDelta` 只含平滑移動量(乘客留在原地自然下落);關卡 JSON 改為單一豎井 y=256 → -64(`moveTiles: 20`,瞬移都發生在畫面外),兩台以 `startOffsetTiles: 10` 等距 160px,視覺上即為原版式的無限輸送帶。
  - **水平平台同相無法跳越**:新增 JSON 欄位 `startOffsetTiles`(沿行程方向的初始偏移格數),讓同組平台錯開相位。1-3 的 x=1504 平台已設 `startOffsetTiles: 6` 與 x=1376 反相;注意兩台 `moveTiles` 不同(5 vs 6)週期不同,相位會緩慢漂移,若要永久反相請把兩台 `moveTiles` 調成一致。

### C-7 🟡 Koopa 殼甦醒無預警,可能在玩家站上面/旁邊時瞬間還原
- [Koopa.cpp:88-103](src/Koopa.cpp#L88):5 秒到直接變回行走狀態,沒有原版的「殼抖動警告動畫」;若玩家正壓著殼,會在同一格瞬間重疊受傷。

### C-8 🟡 KoopaParatroopa(飛行龜)的浮點數相等判斷
- [KoopaParatroopa.cpp:30-32](src/KoopaParatroopa.cpp#L30):Hop 模式用 `if (m_Velocity.y == 0.0f)` 決定重新起跳,依賴碰撞解析剛好把 vy 設為精確 0;任何改動(如改用衰減)都會讓它永遠不再跳。

### C-9 🟡 Player 建構子的初始狀態不一致

### C-10 🟡 主題(Theme)用檔名子字串推導

### C-11 🟡 食人花素材寫死地下版本
